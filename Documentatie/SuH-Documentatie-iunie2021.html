1 Abstract

SuperHeroes Web Game reprezintă un joc educativ bazat pe un set de informații despre super-eroi, asemănător jocului de tip triviador (știi și cștigi), destinat n mod special copiilor și adolescenților. Jocul a fost integrat ntr-un site Web pentru ca un număr ct mai mare de jucători să poată avea acces la el indiferent de tipul dispozitivului folosit.

 

2 Introducere

Pentru realizarea n totalitate a proiectului echipei am folosit următoarele tehnologii: HTML5, CSS3, PHP, JavaScript, MySQL și JSON. SuperHeroes Web Game (SuH) este o aplicaţie Web care, pe lngă jocul n sine, oferă posibilitatea unui jucător care s-a conectat să vizualizeze:

        regulamentul jocului;

        clasamentul general cu ordinea jucătorilor și clasamentul pe țări;

        galeria cu imagini de super-eroi.

        profilul utilizatorului

        date de contact

 

3 Tehnologiile utilizate

3.1 Descrierea tehnologiilor utilizate

HTML Pnă n anul 1990 accesarea informațiilor de pe INTERNET era foarte dificilă și rețeaua nu era folosită dect de un număr restrns de persoane, n general oameni de știință. Fizicianul Tim Berners-Lee a devenit celebru deoarece a inventat link-urile hypertext. Această idee a dus la dezvoltarea unui limbaj simplu care s-a impus pe piața mondială. Acest limbaj a fost numit HyperText Markup Language, prescurtat, HTML.

Astfel, o pagină Web marcată cu tag-uri HTML are forma generală:

<html> -> tag obligatoriu

<head> -> nceput antet

  

   </head> -> sfrşit antet

<body> -> nceput corp

</body> -> sfrşit corp

</html> -> sfrşit document

 

n cadrul antetului, printre altele, poate apărea titlul documentului (al paginii Web) specificat de tag-ul <title>:

<title> SuperHeroes Web Game </title>

De obicei, ntre <head> şi </head> apar definiţii de rutine scrise n limbaje script (ca JavaScript) pentru a fi siguri că sunt analizate și memorate de navigator (antetul e procesat primul).

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile folosite le-am atașat elementelor HTML prin intermediul unor fișiere externe sau n cadrul documentului, prin elementul <style> și/sau atributul style. CSS se poate utiliza și pentru formatarea elementelor XHTML, XML și SVGL. CSS3 reprezintă un upgrade ce aduce cteva atribute noi și ajută la dezvoltarea noilor concepte n Webdesign. Astfel, CSS3 vine cu posibilitatea designului responsive (CSS nu permite acest lucru), poate fi mpărțit n mai multe module și crește viteza de ncărcare a paginii. Stilurile folosite le-am validat cu ajutorul jigsaw.w3.org/css-validator/, recomandat n cerințe, pentru a nu avea erori.

PHP este un limbaj de programare, iar PHP provine din limba engleză și este un acronim recursiv: PHP Hypertext Preprocessor. Folosit inițial pentru a produce pagini Web dinamice, este utilizat pe scară largă n dezvoltarea paginilor și aplicațiilor Web. Se folosește n principal nglobat n codul HTML, dar ncepnd de la versiunea 4.3.0 se poate folosi și n mod linie de comandă (CLI), permițnd crearea unor aplicații independente. Este unul din cele mai importante limbaje de programare Web open-source și server-side, existnd versiuni disponibile pentru majoritatea Web serverelor și pentru toate sistemele de operare. n cadrul proiectului l-am folosit pentru că ne-a asigurat posibilitatea conectării la baza de date, afișarea barei de navigare și a footerului și utilizarea JavaScripturilor pentru rularea anumitor funcții.

JavaScript (JS) limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este utilizat n mod special pentru a introduce anumite funcționalități n paginile web, browser-ul fiind cel care rulează codul JavaScript. Am folosit JavaScript pentru folosirea tastaturii n locul mouseului, traducerea paginilor n romnă și engleză, afișarea profilului cu informații din baza de date, rularea efectivă a jocului cu 3 nivele de experiență (ncepător, intermediar și avansat).

JSON (JavaScript Object Notation) este un format de schimb de date. Se bazează pe un subset al Standardului de limbaj de programare JavaScript ECMA-262. JSON este un format text care este complet independent de limbă, dar folosește convenții familiare programatorilor care utilizează C, C ++, C #, Java, JavaScript, Perl, Python și multe altele. Noi i-am folosit proprietățile pentru a apela ntrebările și răspunsurile din cadrul quizzurilor pe nivele de experiență

Asynchronous JavaScript And XML (AJAX) este folosit ca tehnică de programare pentru a crea pagini web ușor de accesat de către browsere și utilizează mai multe tipuri de tehnici:

1. Obiect de tipul XMLHttpRequest integrat n browser (pentru a cere date de la serverul web);

2. JavaScript și HTML (pentru a afișa sau utiliza datele).

Evenimente care au loc la nivelul AJAX:

1. Un eveniment apare ntr-o pagină web (de exemplu, pagina este ncărcată, se face clic pe un buton);

2. Un obiect XMLHttpRequest este creat de JavaScript;

3. Obiectul XMLHttpRequest trimite o cerere către server-ul web;

4. Serverul procesează cererea;

5. Serverul trimite un răspuns napoi la pagina web;

6. Răspunsul este citit de către JavaScript;

7. Acțiunea (de exemplu, actualizarea paginii) este efectuată de JavaScript.

Tehnologia AJAX am folosit-o n toate etapele proiectului pentru a lega ntre ele diversele module separate, precum și la nregistrările și interogările n baza de date.

Comma Separated Values (CSV) este un fișier text simplu care conține o listă de date. Aceste fișiere sunt adesea utilizate pentru schimbul de date ntre diferite aplicații. De exemplu, bazele de date și managerii de contacte acceptă adesea fișiere CSV. Le-am folosit pentru a importa datele despre super-eroi pe care le-am implementat n cadrul ntrebărilor și răspunsurilor din quizz.

MySQL am folosit o bază de date gestionată cu ajutorul PHPmyadmin, pentru stocarea și accesarea datelor utilizatorilor (ID, usermane, parola, e-mail, țara, puncte acumulate n joc), informații despre super-eroi, generarea clasamentelor. Pentru a evita SQL injection am utilizat parametri care să combine un conținut dinamic n expresiile SQL precum un sistem de criptare a parolelor de acces pentru a evita nterceptarea acestora cu ajutorul "hash".

API l-am folosit pentru a afișa harta Google cu locația utilizatorului. De asemenea, pentru traducerea automata a paginilor n alte limbi dect cele folosite n internaționalizarea siteului (romnă și engleză), am implementat și un API de la Google Translate cu 5 limbi, care poate fi updatat și cu alte limbi. Vom mai folosi la dezvoltare și un API cu un cod reCAPTCHA pentru validarea contului și evitarea creării conturilor false generate automat.

 

 

4 Arhitectura modulelor aplicaţiei

Schema logică generală

 

4.1. Navbar (bara de navigare)

Bara de navigare asigură trecerea ntre diversele module ale siteului.

Conține:Acasă (pentru revenirea la pagina principală), Logare (pentru nregistrarea utilizatorilor), Joacă! (pentru modulul de joc efectiv), Galerie (cu imagini si numele supereroilor), Clasament (cu numele si punctele acumulate in joc), Contact (cu datele de contact ale administratorilor siteului), Despre (o descriere pe scurt a siteului), Regulament (cu 6 reguli principale de utilizare a jocului si trimitere la regulamentul complet).

Sub bara de navigare am implementat funcția "Language option" cu ajutorul căreia utilizatorii pot comuta pagina ntre limbile romnă si engleză.

 

Schema modului de logare a utilizatorilor

4.2 Login

n situația n care un utilizator are un cont existent (n baza de date) atunci acesta va avea posibilitatea de a se loga pe site cu ajutorul user-ului și al parolei aferente.

Parola va fi criptată și stocată astfel n baza de date pentru a evita SQL injection, iar introducerea ei pe terminal se va face n modul Hide (parolă codată) cu minim 8 caractere. Un utilizator nelogat (guest) va avea acces la funcțiile siteului, inclusiv la joc, dar rezultatele nu i vor fi nregistrate n baza de date și nici nu va apărea in clasamente.

Validarea contului se face n baza de date cu ajutorul usernameului și al parolei.

Dacă utilizatorul nu are cont, n fereastra de logare are opțiunea de "Crează un cont" care l va direcționa la pagina de "nscriere".

Dacă acesta nu-și mai amintește parola, tot n fereastra de logare are opțiunea "Ați uitat parola?" care l va direcționa la pagina "Recuperează-ți contul".

4.3 Sign-up

n cazul n care un utilizator nu poate fi corelat cu un user din baza de date a site-ului, atunci acesta va trebui să și creeze un cont nou n care să introducă: e-mail-ul, user-ul, parola (cu confirmare și minim 8 caractere) și țara.

4.4 Forgot password

n cazul n care utilzatorul nu-și mai amintește parola, opțiunea "Ați uitat parola?" care l va direcționa la pagina "Recuperează-ți contul", unde va introduce adresa de e-mail sau numele de utilizator și va primi pe e-mail o parolă generată automat.

4.5 Play Game

Utilizatorii care aleg opțiunea de a juca, vor avea posibilitatea să și pună n valoare cunoștințele despre supereroi folosind unul dintre cele 3 nivele de dificultate: ncepător, intermediar și avansat.

Durata fiecărui quizz este de 60 secunde, iar acesta se termină automat la finalul timpului alocat.

Punctele se acordă astfel:

        pentru nivelul ncepător se acordă 3 puncte la fiecare răspuns corect;

        pentru nivelul intermediar se acordă 4 puncte la fiecare răspuns corect;

        pentru nivelul expert se acordă 5 puncte la fiecare răspuns corect.

Răspunsurile greșite nu sunt punctate și afișează automat răspunsul corect.

Punctele adunate de fiecare user se nregistrează n baza de date și pe baza lor se generează clasamentul general și pe țări.

 

4.6 Rankings

Utilizatorii care și doresc să parcurgă clasamentul general și locul acestora n cadrul clasamentului, vor putea accesa pagina dedicată acestui concept, denumită intuitiv Clasamente.

Pe baza punctelor acumulate vor fi afișate 2 tabele, unul cu clasamentul general și unul cu clasamentul pe țări.

4.7 Galery

n situația n care un utilizator vrea să-și actualizeze informațiile despre supereroi acesta o poate face accesnd pagina Galerie care conține detaliile necesare n acest sens.

 

4.8 Profile

Dacă un utilizator dorește să afle informații despre profilul acestuia (username, nivel, experiență, punctele acumulate, țara setată la logare, orașul de reședință) acesta va utiliza pagina Profile existentă n bara de navigare pentru a avea acces la informațiile dorite.

Dacă utilizatorul nu este logat, nu are acces la pagina de profil.

4.9. Rules

Pentru a putea fi la curent cu regulamentul jocului și pentru a avea n evidență toate detaliile necesare utilizatorul va putea citi 6 reguli importante n pagina dedicată acestora (Regulament).

Tot n cadrul acestei pagini utilizatorii vor putea accesa regulamentul și ghidul de utilizare complet att n format PDF, ct și n format HTML.

4.10. Control panel

Am prevăzut un Control Panel pentru administrarea siteului n backend. Acesta este accesibil doar administratorilor și are acces la anumite funcții legate la baza de date. Asftel se pot șterge utilizatori, supereroi, se pot restricționa anumite drepturi sau se pot vedea clasamentele.

 

5. Descrierea modului de implementare

Pentru realizarea siteului am folosit MVC att pentru front-end ct și pentru back-end. Asftel am adaptat soluțiile pentru fiecare sistem n parte.

6. Internaționalizarea

Proiectul este disponibil n romnă ct n engleză, iar utilizatorii vor putea alege limba n care să fie afișate informațiile din fiecare pagină. Pentru aceasta am folosit un script de traducere automată implementat n codul paginilor.

8. Accesibilitatea

1) Responsive: siteul este creat n mod responsive pentru mai multe rezoluții uzuale ale telefoanelor. Am implementat rezoluțiile standard: 320 px, 375 px, 425 px, 768 px, 1024 px, pentru fiecare pagină din cadrul proiectului.

2) Keyboard: pentru utlizatorii care nu pot folosi mouseul am implementat un script de folosire a tastaturii n locul mouseului cu ajutorul tastelor TAB, SHIFT și ENTER.

3) Descrierea imaginilor: pentru utlizatorii care nu pot vedea imaginile am implementat o descriere text a imaginilor care va apărea n locul acestora.

4) Accesibilitate socială: pentru utilizatorii cu anumite dizabilități sau cu abilități mai reduse n utilizarea internetului, vom implementa un API pentru text-to-spech care să ofere un suport vocal.
